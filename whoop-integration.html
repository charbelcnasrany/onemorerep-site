OneMoreRep × WHOOP — Integration UX

Prepared for WHOOP app review · OneMoreRep is a private social fitness app: users check in at a gym (GPS + Google Places verified) and friends they explicitly added cheer them on.

Why WHOOP

WHOOP workout data is used for exactly one thing: confirming that a recorded workout happened around the time of a gym check-in. A confirmed check-in shows the same verification badge with the source noted ("Verified · WHOOP"). It grants no extra rewards or status — WHOOP data confirms, it never elevates one user above another.

  1. User taps "Connect WHOOP" → standard WHOOP OAuth page opens in the browser (scopes: read:workout, read:profile, offline).
  2. Tokens are exchanged and stored server-side only (Supabase edge function + locked table). The mobile app never sees or stores tokens.
  3. Disconnecting is one tap and deletes our stored tokens immediately.

Privacy & security

Tokens live exclusively server-side (never on-device) · workout confirmation is computed server-side and cannot be spoofed by the client · data is visible only to friends the user explicitly added · disconnect deletes tokens instantly · webhook signatures verified (HMAC) · full policy: privacy policy.
